# v0.9 Console Terminal QA

This document records the terminal QA evidence for issue `#261` and the role
avatar extension in `#299`. The goal is to prove that the K9s-style CoreRoom
console remains a clear engineering control surface rather than a noisy
multi-agent transcript.

## Scope

Validated surfaces:

- overview console render
- active view renders: roles, WorkOrders, gates, evidence, sources, logs, Xray
- public `@user <-> @host` transcript projection
- host action permission overlay
- deterministic README image regeneration

Out of scope:

- live polling from GitHub or engine subprocesses
- release publishing
- mutating console actions
- replacing the normal `cr start` REPL entry

## Terminal Fixtures

The ratatui test backend renders the console at the supported QA widths:

| Width | Height | Purpose |
| --- | --- | --- |
| 80 | 30 | narrow terminal / split pane fallback |
| 120 | 40 | compact desktop terminal |
| 160 | 48 | standard full-screen terminal |
| 220 | 54 | wide K9s-style control room |

The automated fixture checks:

- rendered lines do not exceed the terminal width
- the console is not blank or suspiciously sparse
- the overview keeps the public `Transcript` panel visible in read-only console mode
- specialist Xray/internal activity does not leak into the public transcript
- active detail views do not duplicate the public transcript
- the host action overlay displays `CONFIRMATION REQUIRED`, action kind, and
  non-executable status before any mutation

## README Images

README images are regenerated with:

```bash
make readme-images
```

Generated assets:

- `docs/images/boot-dashboard.png`
- `docs/images/work-cards.png`
- `docs/images/control-room-console.png`

The console image now represents the v0.9 full-screen control surface:
project/repo/branch/phase/tracker anchors, `@host` as the user-facing control
authority, role lanes as side-rail state, WorkOrder/evidence closure, host
action semantics, and tracker gaps that must be resolved before completion.

For `#299`, the image renderer also shows the Role Avatar System. The
documented default remains terminal-safe geometric avatars. Nerd Font avatars
are a rich opt-in path through `COREROOM_AVATAR_PACK=nerd-font`; they are not
required for the README images or the default console.

## Validation Commands

Recorded for `#261` and extended for `#299`:

```bash
cargo test --test console_terminal_qa_test --quiet
make readme-images
```

Full PR validation also runs:

```bash
cargo fmt --all -- --check
cargo test --all-features --locked --quiet
cargo clippy --all-targets --all-features -- -D warnings
git diff --check
```

## Manual Visual Notes

- The center conversation remains the primary surface; internal delegation stays
  in the right rail, Xray, or log views unless the user directly addresses a
  specialist role.
- The right rail follows the Codex-style side panel pattern: progress,
  environment, changes, active roles, evidence, sources, and alerts.
- The host action overlay is centered and explicit. It names the action and
  confirmation requirement instead of letting the console mutate project state.
- README visuals are synthetic but deterministic; they are intentionally tied to
  the current v0.9 console story rather than stale v0.7 release screenshots.

## Rollback

If the QA evidence proves misleading, revert the PR that updates the terminal
fixtures, README image renderer, generated images, and this document. The change
does not alter persisted user state or release artifacts.
