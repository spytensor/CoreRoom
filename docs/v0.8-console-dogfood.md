## v0.9 Readiness Criteria

v0.9 may start only after these v0.8 facts are true:

- `CoreRoomSnapshot` validates project, runtime, roles, work, gates, evidence,
  sources, GitHub facts, alerts, and layout hints.
- public conversation remains `User <-> @host`; specialist roles remain
  internal/side-rail/Xray unless explicitly surfaced by the user or host.
- health selectors expose only actionable signals.
- responsive layout decisions preserve the conversation center panel.
- the README console preview is generated from the snapshot fixture where
  practical and clearly labels itself as a preview.
- tracker `#238` has `#251` and `M-AC-10` checked with Evidence Ledger proof.

## Remaining v0.9 Risks

- The ratatui renderer still needs keyboard focus, resize, and scroll testing
  across 120, 160, and 220+ column widths.
- Live polling must not become a second source of authority. GitHub, Evidence
  Packets, WorkOrders, gates, and source registry state remain the facts.
- The console must not mutate project state directly. Mutating actions route
  through `@host` confirmation and existing GitHub/gate/evidence flows.
- External source refresh and role-session freshness must stay loud when stale.

## Rollback

If the v0.8 dogfood packet proves misleading, revert the PR that added this
document and its fixtures/tests. This does not affect runtime behavior or
persisted user state.
