# v0.9.4 Live Room Dogfood Archive

This document is historical evidence for issue `#306`. The staged live-room
preview it described was removed by `#330` because the bridge mutated an
in-memory snapshot from the input loop instead of projecting real runtime
events.

## Current Status

`cr console --live-room` no longer opens a PTY composer path. It exits with:

```text
cr console --live-room: full-screen runtime is being rebuilt — see #320
```

The replacement full-screen runtime shell is tracked separately. This archive
must not be used as acceptance evidence for that replacement.

## Dogfood Harness

The v0.9 dogfood script no longer types into the removed live-room preview or
asserts staged routing text. It now checks:

- plain `cr` reaches the executable runtime entrypoint
- `cr console` and `cr console --snapshot` remain read-only console paths
- `cr console --live-room` fails with the rebuild notice above

Replacement-specific PTY assertions belong to the TUI-shell issue, not this
archive.

## Historical Scope

The removed scenario used to open:

```text
cr console --live-room
```

inside a 160x48 PTY from a freshly initialized temporary project, then typed
bare user text, an explicit `@reviewer` request, `/journal reviewer`, and
`/exit`. That evidence only proved the staged bridge rendered input and status
inside a terminal. It did not prove role-engine execution, streaming runtime
events, durable turn state, or permission prompt parity.

## Rollback

Revert the PR that removed `src/console_room.rs` and restored the previous
`cr console --live-room` routing if the staged bridge must be recovered. That
rollback would also need to restore the deleted tests and dogfood assertions.
